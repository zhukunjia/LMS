# 图书管理系统功能设计

## 用户模块设计
- 登录协议使用`token`，没有使用`JWT`。真实的系统建议还是使用`JWT` 或者其他方式
- 登录接口返回 `userId` 和 `token`，调其他接口时，请求头需要带上这两个`x-lms-userId` 和 `x-lms-token`
- 这是一个单机系统，缓存使用了内存，没有使用分布式缓存，例如Redis。使用`caffeine` 包，设置过期时间

## 权限设计
- 引入权限组，只有两个 `USER` 和 `MANAGE`，用户可以加入权限组，用户和权限组的关联关系放在单独的表里
- 图书管理模块的接口，会有一个注解，可以传入多个角色
- 理论上需要加一个资源表、资源-权限组关联关系表。当前没有做到这个程度
